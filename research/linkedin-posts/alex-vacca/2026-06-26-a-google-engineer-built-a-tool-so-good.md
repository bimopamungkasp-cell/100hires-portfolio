A Google engineer built a tool so good it hit #1 on Hacker News in two days.

Google fired him for it.

Justin Poehnelt shipped the Google Workspace CLI. Thousands of GitHub stars, thousands of real users, all in a couple of days. Two months later he was out, and his read is that the tool is the reason.

Everyone's talking about the firing as another big-company-eats-its-builders story. But that's not the interesting part to me.

The interesting part is why it blew up.

It was built for AI agents to run. One command line, every Workspace API, 40+ agent skills inside it, I don't have to read Google API docs anymore.

We run outbound for 100+ AI companies, and the same thing is happening to the GTM stack right now. The tools we keep are the ones an agent can run on its own. It pulls the list, writes the variations, sends, and nobody opens a dashboard to make it happen.

The tools that still need a person clicking through five screens to get anything done, we drop them.

So if you're building an AI product, the question isn't whether the interface is nice anymore. It's whether an agent can run your product without reading your docs. Because more and more, the user on the other side is an agent, not a person.

The guy who figured that out first just got walked out of Google.

Funny how that works.
