Waterfall enrichment in Claude Code has arrived, and it has the potential to change the way we do sourcing. 

The team has been playing around with Deepline for the past few weeks, and it has the potential to be a great sourcing plugin for GTM Engineers working in Claude. 

The concept is pretty simple- you prompt Claude to source a list of companies or people in natural language, and Deepline puts the list together for you. 

What’s more interesting is that you can then enrich those leads for basic data points like phone numbers and emails using several different providers. 

What’s even MORE interesting is that it uses tools like Exa to perform agentic research tasks and write those to the table. 

You can then view it in a table format via their session id, which makes it easy to export, or you can just use their native Snowflake/Salesforce/HubSpot connections to write the information there. 

GTM is getting more and more agentic- curious as to whether people have used Deepline or similar tech in their sourcing yet!
